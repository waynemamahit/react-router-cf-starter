# Project Context

## Purpose
- Create a React application with React Router framework mode

## Tech Stack
- Node.js 24 latest LTS
- React 19
- TypeScript 5 latest
- TailwindCSS 4
- React Router latest (Framework Mode)

## Project Conventions

### Code Style
- Use TypeScript for type safety and better code quality
- Use Biome formatting and linting
- Use Tailwind CSS for styling
- Use React Router framework mode for routing

### Architecture Patterns
- Use React Router framework mode for routing
- Use Hono for API routing
- Use React patterns for components, hooks, and state management.

### Testing Strategy
- Use Vitest for unit testing includes backend API and frontend components
- Use React Testing Library for component testing with proper test coverage
- Ensure comprehensive test coverage for all components and API endpoints
- Maintain test coverage above 85% for both frontend and backend

### Git Workflow
- Use Git for version control
- Follow conventional commits format

## Domain Context
- This project is a general-purpose React application built with React Router framework mode for web development, following modern React latest best practices.

## Important Constraints
- Don't use `any` types
- Make sure all types are defined in separate models (class, interfaces and types) in one file.
- Group related models together in their respective files.
- Export all models from a single index file for easy imports.
- Refactor by following clean architecture and SOLID principles.
- Create responsive design and choose best colour based on color harmony and increase accessibility.
- Use semantic HTML and ARIA roles for better accessibility.

## External Dependencies
- None
